How to use GitHub you idiot

  1. Open Command Line program
  2. Change into the directory you're working in.
  3. Login to GitHub
  4. Create a new repository, use the same name as the directory you're working in
  5. Copy and paste the instructions that GitHub gives you into the Command Line
  6. After copying and pasting everything, type the following into the Command Line:
  7. git init (git starts tracking)
  8. git add . (adds all files to staging area)
  9. git commit -m '(add information here)' (commits changes)
  10. git push (sends files off to github)

Git Command Line Commands

  • git init creates a new Git repository
  • git status inspects the contents of the working directory and staging area
  • git add . adds ALL files from the working directory to the staging area
  • git add [filename_1] [filename_2] adds one or multiple files from the working directory to the staging area
  • git diff shows the difference between the working directory and the staging area
  • git commit -m '[message]' permanently stores file changes from the staging area in the repository with a message
  • git log shows a list of all previous commits
  • git show head will show you the recently made commit
  • git checkout HEAD [filename] will restore the file in your working directory to look exactly as it did when you last made a commit.
  • git reset HEAD [filename] resets a file in the staging area to be the same as the HEAD commit
  • git reset [commit_SHA] allows you to rewind to a specific commit. You can view all the commits with sha numbers using git log. You only need the first 7 alphanumeric characters.
